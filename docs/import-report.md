# Import Report

`import_report.json` records what happened when a builder or importer processed sources into an OKPF package. It is optional, generated by tools, and useful for partial success, indexing summaries, and clear failure messages.

## Compatibility Shape

```json
{
  "import_id": "lumina-import-001",
  "status": "partial_success",
  "started_at": "2026-05-12T12:00:00Z",
  "completed_at": "2026-05-12T12:01:00Z",
  "summary": {
    "sources_total": 2,
    "sources_succeeded": 1,
    "sources_failed": 1,
    "records_created": 3
  },
  "items": [
    {
      "path": "sources/example.md",
      "stage": "normalization",
      "status": "success",
      "message": "Created normalized records."
    },
    {
      "path": "sources/bad.pdf",
      "stage": "extraction",
      "status": "error",
      "message": "PDF text extraction failed."
    }
  ]
}
```

`status` may be `success`, `partial_success`, or `failed`. Item stages should use clear practical names such as `discovery`, `container`, `extraction`, `normalization`, `validation`, `chunking`, or `indexing`.

## Source File Shape

Importers may also report per-file counts and errors:

```json
{
  "importer": {
    "name": "example-importer",
    "version": "0.1.0"
  },
  "source_files": [
    {
      "path": "records/wine-recipes.json",
      "status": "indexed",
      "record_count": 41,
      "chunk_count": 273,
      "indexed_count": 273,
      "indexed_unit": "chunk",
      "skipped_count": 0,
      "failed_count": 0,
      "record_type_counts": {
        "recipe": 41
      },
      "facet_counts": {
        "beverage_type": {
          "wine": 41
        }
      }
    }
  ],
  "errors": [
    {
      "source_file": "records/example.json",
      "stage": "index",
      "message": "Example sanitized error message"
    }
  ]
}
```

`record_count` is the count of normalized source records.

`chunk_count` is the count of derived chunks generated for retrieval, indexing, display, or other consumer workflows.

`indexed_count` is the count of indexed chunks or indexed units, depending on the importer. If `indexed_count` is tool-specific, the report should define the counting unit with `indexed_unit`.

`record_type_counts` and `facet_counts` are optional summaries that let consumers display counts such as records by type or records by beverage without reparsing all record files.

ZIP-aware importers may include `parent_path` or `container_path` to describe parent/child files inside archives.
